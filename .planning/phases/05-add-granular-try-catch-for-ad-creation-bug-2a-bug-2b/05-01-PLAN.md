<objective>
Wrap each ad creation call (metaAPI.createAd) in an individual try-catch inside the creative loop, so that one failed ad does not kill the remaining creatives or bubble up to the adset-level catch. Track adsCreated/adsFailed in the existing statsPerCampaign object. Add DB insert error checking for ads.

Purpose: Currently, if metaAPI.createAd throws for one creative, the error is caught by the outer adset try-catch, which increments adsetsFailed (wrong counter) and skips all remaining creatives for that adset. This phase isolates ad failures so the loop continues to the next creative.

Output: Modified bulk-publish.ts with per-ad try-catch, stats tracking, and DB error checking.
</objective>

Inside the `for (const creativeName of validCreatives)` loop (currently starting around line 242), wrap the entire body of each iteration in its own try-catch. The changes are:

1. **Add try-catch around each creative iteration** — The `try` block starts right after `for (const creativeName of validCreatives) {` and the `catch` block goes at the end of the loop body. The catch clause uses `(adErr: any)` and does:
   - `console.error(\`[bulk-publish] Ad creation failed for creative "${creativeName}" in adset ${metaAdsetId} campaign ${metaCampaignId}:\`, adErr?.message || adErr);`
   - `statsPerCampaign.adsFailed++;`
   - `continue;`

2. **Add `statsPerCampaign.adsCreated++`** — After the successful `metaAPI.createAd` call and its DB insert, increment the counter. Place it after the DB insert block (after the `if (storedAccount)` block for meta_ads insert), before `await humanDelay()` (which is currently at line 301).

3. **Add DB insert error check for ads** — The current code (lines 303-312) does:
   ```typescript
   await supabase.from('meta_ads').insert({...});
   ```
   Change this to:
   ```typescript
   const { error: dbAdErr } = await supabase.from('meta_ads').insert({...});
   if (dbAdErr) {
     console.error(`[bulk-publish] DB insert ad failed for adset ${metaAdsetId}, creative "${creativeName}":`, dbAdErr);
   }
   ```

4. **Move `await humanDelay()`** — Currently at line 301 (right after createAd). Move it inside the try block, after the DB insert and adsCreated increment. This way, if createAd throws, we skip the delay too.

The resulting loop structure should be:
```typescript
for (const creativeName of validCreatives) {
  try {
    // ... existing URL resolution, video skip, UTM params, adBody construction ...

    const adResult = await metaAPI.createAd(metaAccountId, metaAdsetId, adBody, user.id);

    if (storedAccount && adResult?.id) {
      const { error: dbAdErr } = await supabase.from('meta_ads').insert({
        meta_account_id: (storedAccount as any).id,
        adset_id: metaAdsetId,
        ad_id: adResult.id,
        name: adsetName,
        status: adsetType.adsetStatus,
        creative_spec: adBody.creative,
        last_synced: new Date(),
      } as any);
      if (dbAdErr) {
        console.error(`[bulk-publish] DB insert ad failed for adset ${metaAdsetId}, creative "${creativeName}":`, dbAdErr);
      }
    }

    statsPerCampaign.adsCreated++;
    await humanDelay();
  } catch (adErr: any) {
    console.error(
      `[bulk-publish] Ad creation failed for creative "${creativeName}" in adset ${metaAdsetId} campaign ${metaCampaignId}:`,
      adErr?.message || adErr
    );
    statsPerCampaign.adsFailed++;
    continue;
  }
}
```

IMPORTANT notes:
- The `if (!rawUrl) continue;` and `if (creativeFile?.type === 'video') continue;` guards at the top of the loop body should remain OUTSIDE the try block (before it) — they are pre-filters, not operations that can fail.
- The `adResult` variable must use optional chaining: `adResult?.id` (not bang operator `adResult!.id`). Phase 04 established this pattern.
- Do NOT move the ad loop out of the adset try-catch block — it stays nested inside. The ad-level try-catch is a SECOND inner layer.
- The `const creativeFile = creativeFiles.find(...)` and URL resolution code stays inside try since it accesses variables that could theoretically be undefined.
