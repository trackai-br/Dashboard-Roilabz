  <action>
Add a new top-level async function `verifyCampaignStructure` BEFORE the `handler` export (around line 31, after `uniqueSuffix`). This function:

1. Accepts `(metaCampaignId: string, userId: string)` — note: no accountId needed, Meta Graph API queries by campaign ID directly.

2. Calls `metaAPI.getAdSets(metaCampaignId, ['id'], 100, undefined, userId)` to get all adsets under the campaign.

3. For each adset returned, calls `metaAPI.getAds(adset.id, ['id'], 100, undefined, userId)` to count ads.

4. Computes total adCount across all adsets.

5. Determines status:
   - `'complete'` if adsetCount > 0 AND adCount > 0
   - `'partial'` if adsetCount > 0 BUT adCount === 0 (adsets exist but no ads)
   - `'empty'` if adsetCount === 0

6. If status is `'empty'`, log: `[bulk-publish] [verification] WARNING: Campaign ${metaCampaignId} has no adsets — verification status: empty`

7. If status is `'partial'`, log: `[bulk-publish] [verification] WARNING: Campaign ${metaCampaignId} has ${adsetCount} adsets but 0 ads — verification status: partial`

8. Returns typed object:
```typescript
{ campaignId: string; adsetCount: number; adCount: number; status: 'complete' | 'partial' | 'empty' }
```

9. The ENTIRE function body is wrapped in try-catch. On error, log: `[bulk-publish] [verification] Verification failed for campaign ${metaCampaignId}: ${error.message}` and return `{ campaignId: metaCampaignId, adsetCount: -1, adCount: -1, status: 'empty' as const }` — this ensures verification errors never fail the campaign.

Important: Use minimal fields (['id'] only) in getAdSets/getAds calls to minimize API load and rate limit risk. Add a small delay (200ms) between getAds calls to respect Meta rate limits — use the existing `delay()` function at line 17.
