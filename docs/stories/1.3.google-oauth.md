# Story 1.3: Google OAuth — Autenticação & Google Ads Sync

## 📌 Overview

**Epic:** 1 — Core Sync & Dashboard Básico
**Story ID:** 1.3
**Complexity:** Large (L)
**Estimated Points:** 16
**Status:** 🟡 Draft

---

## 📖 User Story

> Como **Account Manager**, quero **autenticar com Google OAuth** para integrar minhas contas **Google Ads**, para **gerenciar campanhas Google e Meta em um único painel**.

---

## ✅ Acceptance Criteria

- [ ] Google OAuth setup com provider (Google Cloud Console)
- [ ] Login/logout functionality via Supabase Auth (OAuth)
- [ ] User profile stored in `users` table (Supabase)
- [ ] Sync Google Ads accounts via `google_ads_sync` Inngest function
- [ ] Table `google_ads_accounts` with account data (customer_id, account_name, etc.)
- [ ] Table `google_ads_campaigns` with campaign data
- [ ] TanStack Query integration with 15min staleTime
- [ ] Protected routes (middleware for auth check)
- [ ] User can disconnect Google account
- [ ] Error handling for OAuth failures (invalid token, revoked access)
- [ ] Lighthouse score ≥ 90 (performance)

---

## 🎯 Implementation Plan

### Phase 1: OAuth Setup & Authentication (5 points)
- [ ] Google Cloud Console: Create OAuth 2.0 credentials
- [ ] Supabase: Enable Google OAuth provider
- [ ] Create `<GoogleAuthButton />` component
- [ ] Create `/login` page with Google OAuth button
- [ ] Auth middleware for protected routes
- [ ] Store user profile in `users` table (on first login)

**Acceptance:**
- Google OAuth credentials configured in `.env.local`
- User can click "Login with Google"
- User redirected to Google consent screen
- After approval, user logged in and profile stored
- User profile visible in Supabase `users` table
- Session persists across page refreshes
- User can logout

### Phase 2: Google Ads Sync (6 points)
- [ ] Google Ads API setup & credentials
- [ ] Inngest function: `syncGoogleAdsAccounts` (similar to Meta)
- [ ] Database migration: `google_ads_accounts` table
- [ ] Database migration: `google_ads_campaigns` table
- [ ] RLS policies for `google_ads_*` tables
- [ ] Hook `useGoogleAdsAccounts()` with TanStack Query
- [ ] Hook `useGoogleAdsAccountSync()` to trigger sync
- [ ] Error boundaries for sync failures

**Acceptance:**
- Cron job syncs Google Ads accounts every 15min
- `google_ads_accounts` table populated with user's accounts
- `google_ads_campaigns` table populated with account campaigns
- RLS: Users see only their accounts
- Sync handles token expiry gracefully (refresh token flow)
- Error boundary shows sync status/errors

### Phase 3: UI Integration & Testing (5 points)
- [ ] Add Google Ads account selector to dashboard
- [ ] Display Google Ads campaigns in main campaign table
- [ ] Account disconnection UI (revoke Google access)
- [ ] Sync status indicator (last synced timestamp)
- [ ] Unit tests for auth hooks + components
- [ ] E2E tests: Login → Google OAuth → Sync → Dashboard
- [ ] Integration tests: Token refresh flow

**Acceptance:**
- Dashboard shows both Meta + Google Ads campaigns
- Can disconnect Google account (revokes API access)
- Last sync timestamp visible to user
- Unit tests >80% coverage
- E2E tests pass: full flow from login to data display
- Lighthouse score ≥ 90
