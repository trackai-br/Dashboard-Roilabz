# Story 1.2: Dashboard Básico — KPI Cards + Campaign Table

## 📌 Overview

**Epic:** 1 — Core Sync & Dashboard Básico
**Story ID:** 1.2
**Complexity:** Large (L)
**Estimated Points:** 16
**Status:** 🟡 Draft

---

## 📖 User Story

> Como **Account Manager**, quero ver **KPIs consolidados** (spend, impressions, clicks, ROAS) em uma tela, com listagem de todas as campanhas, para **tomar decisões rápidas**.

---

## ✅ Acceptance Criteria

- [ ] Layout responsivo com header + sidebar (mobile, tablet, desktop)
- [ ] KPI cards: Total Spend (MTD), Impressions (7d), Avg CPC, Avg ROAS
- [ ] Agregação automática de todas as contas (selector para conta individual)
- [ ] Tabela campanhas com colunas: Account | Name | Status | Spend | Impressions | Clicks | CPC | ROAS | Last Modified
- [ ] TanStack Query com staleTime: 15min
- [ ] Dark mode support (Tailwind CSS)
- [ ] Loading states + error boundaries
- [ ] Lighthouse score ≥ 90 (performance)

---

## 🎯 Implementation Plan

### Phase 1: Layout & Components Setup (3 points)
- [ ] Create `<DashboardLayout />` with header + sidebar
- [ ] Create `<KPICard />` reusable component
- [ ] Create `<CampaignTable />` component
- [ ] Tailwind responsive setup (mobile-first)

**Acceptance:**
- Header: Logo, user profile dropdown, notifications icon
- Sidebar: Navigation links (Dashboard, Insights, Creatives, Settings)
- Components render without errors, responsive on mobile/tablet/desktop

### Phase 2: Data Integration (5 points)
- [ ] Hook `useMetaAccounts()` with TanStack Query
- [ ] Hook `useMetaCampaigns()` with filters/pagination
- [ ] Supabase RLS integration
- [ ] Error boundaries + loading skeletons

**Acceptance:**
- KPI cards fetch from `meta_accounts` table
- Campaign table fetches from `meta_campaigns` table
- RLS enforced: authenticated users see only their accounts
- Loading skeleton shows while data fetches
- Error boundary catches and displays errors gracefully

### Phase 3: Polish & Testing (3 points)
- [ ] Dark mode toggle (Context + localStorage)
- [ ] Lighthouse audit & performance fixes
- [ ] Unit tests (Jest + RTL)
- [ ] E2E tests (Playwright)

**Acceptance:**
- Dark mode persists across sessions
- Lighthouse score ≥ 90
- Unit tests: >80% coverage
- E2E tests pass: login → dashboard → KPIs visible

## 📝 Notes & Considerations

### Design Notes
- Mobile-first approach (agency users on mobile)
- Use existing Tailwind design system (no custom CSS where possible)
- Account selector as dropdown (left sidebar or header)
- KPI cards: 4 columns on desktop, 2 on tablet, 1 on mobile

### Performance Notes
- TanStack Query staleTime: 15min (aligns with Inngest sync)
- Campaign table: Pagination 50 rows/page (default)
- Dark mode: Use Tailwind `dark:` prefix
- Lighthouse: Optimize images, lazy load below-fold components
